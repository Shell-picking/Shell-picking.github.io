cmake的简单使用
cmake是一个工程管理工具，帮助工程师更加方便的编译源代码。在一个cmake工程中，用cmake来生成一个makefile文件。然后用make命令，根据makefile中的内容，来编译整个工程。(cmake和makefile类似于高级语言和汇编语言的关系)。

在工程文件中，编写CMakeLists.txt文件来指定编译过程。告诉cmake我们要对这个目录下的文件做什么事情。
如：

CMakeLists.txt

下面调用cmake对工程进行分析： 

cmake .

cmake会输出一些编译器等信息，然后在当前目录下生成一些中间文件，其中最重要的就是MakeFile。使用再make命令对工程进行编译：

make .

如果顺利通过就可得到helloworld可执行程序。

在这个过程中，cmake生成的中间文件会留在代码文件中。当想要发布代码时，需要把他一个个删除。一种更好的做法是让这些文件都放在一个中间目录中，在编译成功后，把中间文件夹删除即可。所以常见编译cmake工程的做法是：

mkdir build
cd build
cmake ..
make

编译完成后只需要将build文件夹删除即可。
